# 07 — Action Rolls

> Source: Wares Blade Playkit, Chapter 4 (pp. 27–29)

Action Rolls resolve any uncertain attempt. Six types: **SEN, AGI, WIL, CON, CHA, LUC**. (LUC works differently — see below.)

## Procedure

1. **WM declares the Roll type and Difficulty.** If the character has a relevant Skill, the WM should say so.

   **Difficulty examples:**

   | Value | Description |
   |---|---|
   | 1–3 | Even a child could do it. |
   | 4–5 | Easy without the relevant skills. |
   | 6–7 | Easy with the relevant skills. |
   | 8–9 | Average for mavericks; challenging for laymen. |
   | 10–11 | Manageable for skilled professionals; tough without skills. |
   | 12–13 | High risk of failure even for professionals. |
   | 14–15 | Maybe if you're lucky. |
   | 16+ | Typically impossible. |

2. **Player rolls 1D10** and adds the relevant Base Value (and Skill Level if any):

   ```
   Roll Result = Attribute Base Value + Skill Level (if applicable) + 1D10
   ```

   If a Skill Level is **≥ Difficulty**, the player can re-roll any 1 once.

3. **Compare to Difficulty:**
   - `Outcome ≥ Difficulty` → success.
   - `Outcome < Difficulty` → failure.
   - **All dice come up 1** → **Total Failure** (even if the outcome would otherwise succeed).
   - **All dice come up 10 AND outcome ≥ Difficulty** → **Perfect Success**.

4. WM announces the result.

## LUC Rolls

For situations of pure luck (e.g. who the bear charges first). Procedure differs:

- The player decides how much of their Available LUC to spend, rolls that many D10s, and sums them.
- If the sum ≥ Difficulty, success.
- If they don't want to spend any LUC, they can refuse to make the Roll, but must accept any misfortune that produces.
- If Available LUC is already 0, the LUC Roll automatically fails.

## Knowledge Rolls

A special type of LUC Roll. Used to determine whether a maverick happens to know a piece of information that can't be learned through Skills (e.g. coincidentally knowing when a war happened in this region).

Word-Casters get +1 to all Knowledge Rolls (class perk).

## Using LUC in Action Rolls

After Difficulty is declared, the player can spend LUC to add extra D10s — one D10 per LUC point spent — adding all to the original result.

## First Impression Rolls

When two characters meet for the first time, both make a CHA Roll (WM rolls for NPCs):

- Difficulty for each = the other character's Base CHA.
- Each character adds 1D10 to their own Base CHA.
- The character with the higher total has made a positive first impression. Ties go to the PC.

If the difference is **≥ 10**, or either side gets Perfect Success / Total Failure, the reaction is *extreme*. NPCs with the lower total may turn hostile.

Spiritualists: +1 vs same faith, −1 vs all others.

## Perfect Successes and Total Failures

Every Roll has special outcomes for "all 10s" / "all 1s":

- **Perfect Success** bestows extra benefits (Roll-type-specific).
- **Total Failure** imposes special penalties.
- The WM picks something appropriate when rules don't specify.

### Proficiency Gains

Both Perfect Successes and Total Failures generate Proficiency Points in the Skill used:

| Default | Perfect Success | Total Failure |
|---|---|---|
| Default | 10 PP | 5 PP |

(See [Chapter 10](./10-advancement.md) for the impact-modifier table.)

If no Skill was used, the WM picks an appropriate one. If the character doesn't have it, they gain it at Level 0 and earn half PP.

### LUC recovery (Perfect Success only)

A Perfect Success on **anything other than a combat-related Roll** immediately increases Available LUC by 1.

If LUC was spent on the Roll, all bonuses (or penalties) are multiplied by the total number of dice rolled — including PP gain and LUC recovery.

## When you can skip the Roll

- Action is inconsequential.
- Success is obvious.
- Relevant Skill Level ≥ Difficulty.

**Always Roll** for combat-related actions and using Techniques, even if success is guaranteed.
